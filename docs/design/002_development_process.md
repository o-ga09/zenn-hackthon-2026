# 開発プロセス設計

## 概要

AI旅行Vlog自動生成アプリの開発プロセスは、Kiro AIアシスタントによる自動化フックを活用し、実装とドキュメントの同期、品質管理、効率的なPR作成フローを実現している。

## 開発フロー

### 1. ブランチ戦略

```
main
├── feature/[機能名]     # 新機能開発
├── fix/[修正内容]       # バグ修正
├── docs/[ドキュメント]   # ドキュメント更新
└── refactor/[対象]      # リファクタリング
```

### 2. 実装プロセス

1. **ブランチ作成**: `git checkout -b feature/[機能名]`
2. **実装**: コード作成・修正
3. **自動化フック実行**: エージェント完了時に自動トリガー
4. **ドキュメント同期**: 関連ドキュメントの自動更新
5. **PR作成**: GitHub MCP連携による自動ガイド

### 3. 品質管理

#### コミット規約

- **メッセージ形式**: Conventional Commits準拠
- **粒度**: 論理的単位での分割
- **原子性**: 各コミットで動作する状態を維持

#### コードレビュー

- **必須レビュー**: 全PRでコードレビュー実施
- **自動チェック**: リンター、テスト、型チェック
- **ドキュメント確認**: 実装に伴うドキュメント更新の確認

## 自動化フック

### 実装後ドキュメント更新フック

- **目的**: 実装とドキュメントの同期維持
- **対象**: プロジェクトドキュメント、Steeringファイル、README
- **実行タイミング**: エージェント実行完了時

### 開発フロー PR作成フック

- **目的**: 一貫したPR作成プロセスの実現
- **機能**:
  - ブランチ状態確認
  - コミットベストプラクティス適用
  - PR作成テンプレート提供
  - GitHub MCP連携支援

## ツール連携

- **リポジトリ**: `o-ga09/zenn-hackthon-2026`
