# PRテンプレート設計

## 概要

AI旅行Vlog自動生成アプリの開発効率向上とコード品質維持のため、GitHub Pull Requestテンプレートを設計・実装した。

## 設計方針

### 1. 一貫性の確保

- **標準化されたPR構造**: すべてのPRで同じ形式を使用
- **必要情報の網羅**: 変更内容、テスト方法、影響範囲を明確化
- **チェックリスト**: 品質管理項目の確実な実行

### 2. 開発効率の向上

- **テンプレート自動適用**: PR作成時に自動でテンプレートが適用
- **段階的な記入**: セクション別に整理された記入項目
- **レビュー効率化**: レビュアーが確認すべき項目の明確化

### 3. 品質管理の強化

- **コード品質チェック**: リンター、テスト、型チェックの確認
- **ドキュメント同期**: 関連ドキュメント更新の確認
- **セキュリティ確認**: セキュリティ観点でのチェック項目

## テンプレート構造

### 基本情報セクション

```markdown
## 概要

## 変更内容

## 技術詳細

## テスト方法

## 影響範囲

## 関連Issue

## 破壊的変更
```

### 品質管理セクション

```markdown
## チェックリスト

### コード品質

### ドキュメント

### Git

### セキュリティ
```

### レビュー支援セクション

```markdown
## スクリーンショット

## 追加情報

## レビュアー向け情報
```

## 各セクションの詳細

### 概要・変更内容

- **目的**: PRの内容を簡潔に伝える
- **形式**: 箇条書きによる明確な分類
- **カテゴリ**: 新機能、バグ修正、リファクタリング、ドキュメント更新

### 技術詳細

- **目的**: 実装の技術的背景を説明
- **内容**: アーキテクチャ変更、使用技術、設計判断の理由
- **対象**: 技術的なレビューを行うエンジニア

### テスト方法・影響範囲

- **目的**: 変更の検証方法と影響を明確化
- **形式**: ステップバイステップの手順
- **範囲**: フロントエンド、バックエンド、データベース、API別

### チェックリスト

#### コード品質

- リンター・フォーマッター実行
- 型チェック実行
- テスト実行
- コードレビュー実施

#### ドキュメント

- 関連ドキュメント更新
- API仕様書更新
- README.md更新
- Steeringファイル更新

#### Git

- コミットメッセージ規約準拠
- 論理的なコミット分割
- 単一責任の原則

#### セキュリティ

- セキュリティ問題の確認
- 機密情報の除外
- 認証・認可の適切な実装

## 運用ガイドライン

### PR作成者

1. **テンプレート記入**: 各セクションを適切に記入
2. **チェックリスト確認**: すべての項目を確認してチェック
3. **レビュアー配慮**: レビューしやすい情報を提供

### レビュアー

1. **テンプレート確認**: 必要な情報が記載されているか確認
2. **チェックリスト検証**: 品質管理項目が適切に実行されているか確認
3. **動作確認**: 記載された手順で動作確認を実施

## 期待効果

### 開発効率

- **PR作成時間短縮**: テンプレートによる記入項目の明確化
- **レビュー時間短縮**: 必要情報の整理による効率的なレビュー
- **コミュニケーション改善**: 標準化された情報共有

### 品質向上

- **一貫した品質管理**: チェックリストによる確実な品質確認
- **ドキュメント同期**: 実装とドキュメントの同期維持
- **セキュリティ強化**: セキュリティ観点での確認項目

### チーム協力

- **知識共有**: 技術詳細の明確な記載
- **レビュー品質**: レビュアー向け情報による効果的なレビュー
- **新メンバー支援**: 標準化されたプロセスによる学習支援

## 今後の改善

### テンプレート最適化

- **使用状況分析**: テンプレート使用状況の分析
- **フィードバック収集**: 開発者からのフィードバック収集
- **継続的改善**: 使用実績に基づくテンプレート改善

### 自動化連携

- **CI/CD連携**: チェックリスト項目の自動確認
- **品質ゲート**: 必須項目未完了時のマージ防止
- **メトリクス収集**: PR品質メトリクスの収集・分析
